# 深夜树洞 — 产品设计文档

## 产品定位

一款 Chrome 浏览器插件，为深夜陷入懊悔、自我否定情绪的人提供一个安全的情绪出口。它是一个具备心理疏导能力的树洞——平时沉默倾听，在关键时刻温和介入。

## 核心价值

- **打断 rumination 循环** — 物理上把用户从反复浏览的页面拉出来
- **情绪外化** — 把模糊的懊悔拆解成具体的小点，降低恐惧感
- **看见变化** — 通过记录和 AI 总结，让用户看到自己的情绪轨迹

## 用户场景

深夜 12 点到凌晨 6 点，用户因过去的决定陷入懊悔，反复浏览社交媒体、搜索相关内容、或在浏览器中游荡。插件介入，引导用户把情绪说出来、拆解清楚、然后封存。

## 触发机制

### 时间触发

- **守护时段**：00:00 - 06:00
- **触发方式**：时段内，插件图标变色并出现小红点提醒
- **进入方式**：用户点击图标后进入干预页面（非强制弹出）
- **二次提醒**：如果用户不点击，30 分钟后再提醒一次，之后不再打扰

### 手动触发

- 用户随时可以点击插件图标，主动进入干预流程
- 适用于白天或其他时段的情绪低落

## 干预流程

### 第一步：温和提醒

新标签页全屏接管，展示温暖的提醒文字：

> "夜深了，你现在的情绪可能在放大很多事。深呼吸，先停一下。"

底部两个按钮：
- "我想说说" → 进入引导提问
- "我还是要继续" → 10 秒温柔延迟

### 10 秒延迟机制

- 屏幕展示一句温暖的话 + 简短呼吸引导
- 倒计时 10 秒后出现"继续"按钮
- 如果用户点了"继续"，记录"跳过干预"行为
- 放行后不再打扰

### 第二步：写下现在的想法

> "你现在在想什么？"

一个文本框，用户自由书写。不限字数，不催促。

### 第三步：AI 辅助拆解

AI 读取用户写的内容，主动给出几个可能的拆解方向：

> "听起来可能是这几个方面让你难受：
> 1. 你觉得自己说了不该说的话
> 2. 你后悔没有做某件事
> 3. 你害怕失去这个人
>
> 哪个更接近？"

**如果用户描述不清楚：**
- AI 先尝试提供拆解方向让用户选择
- 如果用户还是选不出来，退回到情绪标签选择
- 全程不施压："说不清楚也没关系，随便选一个就行"

### 第四步：情绪标签

提供情绪选项让用户标记当下状态：
- 后悔
- 焦虑
- 孤独
- 愤怒
- 其他（可自定义）

### 第五步：封存仪式

> "今晚的话，我替你收好了。"

- 记录保存到本地
- 温暖但干脆的收尾
- 不说"明天会更好"——不许空头支票

## AI 角色定义

### 身份

树洞。具备心理咨询师能力的树洞。

不是咨询师在倾听，是一个沉默的树洞，但在你快要掉下去的时候，它知道怎么接住你。

### 行为规则

- **话少**。能用一句的不用三句。
- **不评判**，不说"你应该"。
- **帮助拆解时，只是提问，不是引导**。"哪个部分最让你难受？"
- **不假装是人**，不假装有情绪，不假装理解你。你只是一个安全的空间。
- **不说"我理解你"** — 因为你不需要证明你理解，只需要在。

### 何时动用心理疏导能力

树洞的默认状态是沉默。只有在以下时刻才打破沉默：

- 用户连续 3 轮表达强烈的自我否定
- 用户出现自伤相关的表达
- 用户明确说"我不知道该怎么办"

### 疏导方式

- 不是"指出"用户的思维模式，而是通过"提问让用户自己看见"
- 例：用户说"我这辈子完了" → 不说"你这是灾难化思维" → 而是问"你说的'完了'，具体是指什么？"
- 让用户自己拆解，比告诉他"你在灾难化"更有效，也不会触发防御心理

### 危机处理

- 当检测到自伤倾向时，明确表达关心并建议寻求专业帮助
- 提供心理援助热线信息

## 历史回顾

### Popup 入口（简要）

点击插件图标，弹出面板展示：
- 上次记录的时间和情绪标签（一句话摘要）
- 一句随机的温暖短句
- "查看历史"入口链接

### 专用历史页面（详细）

点击"查看历史"打开完整页面，包含：

**时间线**
- 每个条目展示：
  - 时间（如"5月22日 02:34"）
  - 情绪标签（小标签展示）
  - AI 生成的一句话摘要（不展示原文，保护隐私感）
  - 是否跳过干预（小图标标记）

**标签云图**
- 展示哪些情绪出现最多
- 按频率大小可视化

**情绪分布**
- 按周/月的情绪分布图表
- 帮助用户看到情绪变化趋势

**AI 总结洞察**
- 基于历史记录的模式识别
- "你最近三周的焦虑频率在下降"
- "你总是凌晨 2 点跳过干预"

### AI 总结功能

- 调用 AI API 对历史记录进行分析
- 数据只过路处理，不持久化到第三方
- 总结内容存本地

## 首次使用引导

安装插件后的引导流程：

1. **欢迎页** — "这是一个深夜树洞，你的情绪不会被任何人看到。"
2. **隐私承诺** — "所有数据只存在你的浏览器里，AI 只是过路分析，不会存储。"
3. **配置 API Key** — 引导用户输入 DeepSeek API Key
4. **试用邀请** — "要不要现在试一次？"让用户在白天、情绪平稳时先体验一次流程

## 隐私模型

### 数据存储

- 所有情绪记录存储在浏览器本地（IndexedDB）
- 不上传到任何服务器
- 换设备数据丢失（第一版不支持同步）

### AI 调用

- AI 总结通过 API 调用处理，数据不持久化到第三方
- API 调用是"过路"处理，分析完即丢
- 用户需要提供自己的 API Key

### 隐私承诺

- 向用户明确传达：数据不会被第三人知道
- 插件内展示隐私说明

## 技术架构

### 技术栈

| 层面 | 选择 |
|------|------|
| 框架 | React + TypeScript + Vite + CRXJS |
| AI 服务 | DeepSeek（默认），接口可配置 |
| 存储 | IndexedDB，Dexie.js 封装 |
| 后端 | 无，纯客户端 |

```typescript
interface AIConfig {
  baseUrl: string    // 默认 https://api.deepseek.com
  apiKey: string     // 用户提供，加密存储
  model: string      // 默认 deepseek-chat
}
```

## 产品迭代路径

### V1（当前）

- 时间触发 + 手动触发
- 干预流程（五步完整流程）
- 本地存储 + AI 辅助拆解
- 历史回顾（时间线 + 标签云 + 情绪分布 + AI 总结）
- 首次使用引导

### V2

- 关键词触发（检测用户搜索或浏览的关键词）
- 网页内容记录（记录用户反复访问的页面类型）
- 用户可绑定自己的 API Key

### V3

- AI 预测（基于历史数据预测用户可能的 emo 时段）
- 相似问题反馈（当用户提出类似问题时，展示之前的变化）
- 可配置的守护时段
- 云端同步（可选）

## 设计原则

1. **温暖但不腻** — 像一个安静的朋友，不是话痨
2. **引导但不强迫** — 始终给用户"继续"的选项，但增加适度摩擦
3. **具体化模糊的痛苦** — 帮用户把"一切都完了"拆解成"你后悔说了某句话"
4. **隐私是基石** — 用户愿意把最脆弱的时刻交给你，前提是信任
5. **不许空头支票** — 不说"明天会更好"，因为你不知道
